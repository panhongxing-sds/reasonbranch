# Method 候选 —— Confidence-Gated Adaptive-Depth Speculative Decoding

日期: 2026-07-14 · de-risk 数据: `outputs/vsignal/tokens.jsonl`
· 分析: `action_study/sd_audit/run_adaptive_draftlen_derisk.py`

> **一句话**:提出"按校准置信度动态延伸草稿深度"的 method——高置信就往深挖(近乎无损塌缩多次
> target 前向),置信一掉立刻收。**de-risk 在完整 1.5B draft 上给了绿灯**(置信度预测接受
> AUC 0.86、校准极干净、20% 块吃满 cap = 长 run 存在);**但在我们自训的 EAGLE-3 head 上前提
> 不成立**(短跑者,几乎无长 run)。当前状态:**信号真、机制清楚,但未证实能超越 vanilla
> EAGLE-3 → 还不能作为论文主结果**。

---

## 1. 想法与逻辑链

- 单请求 memory-bound,开销 = **target 前向次数**;提速 = 每次前向多吃被接受的 token。
- vanilla EAGLE-3 用**固定深度**草稿链;在推理 CoT 的长易跨度(算术/格式/复述)上,固定深度
  要么白挖(硬块)、要么截断(易块本可吃更多)。
- 若有一个**廉价、校准良好**的 draft 自信号能判断"这一步会不会被接受",就能**逐块不定深度**:
  高置信延伸、低置信收手 → 高接受长度 + 低草稿浪费 → target 前向更少 → 更快。

**新颖点**:不是 EAGLE-2 那种固定节点预算内按概率扩树,而是**按无损校准阈值做每块不定深度延伸**,
并以推理模型"长易跨度"结构为支撑。

## 2. de-risk 结果(完整 1.5B draft,11520 token)

**[1] 置信度预测"是否被接受"的 AUC:**

| 信号 | AUC |
|---|---|
| draft_top1_prob | **0.862** |
| draft_logprob_self | 0.862 |
| draft_margin | 0.851 |
| draft_entropy | 0.846 |

**[2] 按 top1_prob 五分位的 P(accept)(校准极干净、单调):**

| 分位 | top1_prob 区间 | P(accept) |
|---|---|---|
| q0 | 0.08–0.51 | 0.424 |
| q1 | 0.51–0.74 | 0.684 |
| q2 | 0.74–0.95 | 0.872 |
| q3 | 0.95–1.00 | **0.973** |
| q4 | =1.00 | **1.000** |

→ 置信 >0.95 时接受率 0.97,=1.0 时 1.00。**这正是"可以安全地往高置信区深挖"所需**。

**[3] 块结构双峰**:accept_len 直方图 `{0:365, 1:202, 2:215, 3:112, 4:87, 5:80, 6:48, 7:40, 8:291}`
→ **20.2% 的块吃满 cap=8**(会想继续) = 长易跨度真实存在。

**[4] 局限**:数据被 cap 在 8,只能证明"cap 内提前停"没用(白剪),**无法量化真正的收益——在
20% 易跨度上把草稿深挖超过 8**。

## 3. 反证:自训 EAGLE head 上前提不成立

k-sweep(见 `eagle3_drafter_pivot.md` §4):

| drafter | 深层接受 | 长 run |
|---|---|---|
| 1.5B(de-risk 数据) | 衰减慢 | 20.2% 吃满 8 ✅ |
| 自训 EAGLE head(k=8) | 第 8 位 0.027 | 几乎没有 ❌ |

EAGLE head 是**短跑者**:第一步准(0.63)但自回归后迅速失真。**自适应深度延伸的 headroom 来自
长 run,而这个 head 没有长 run** → 方法前提在它上面不成立。很可能是**欠训**(1600 条/2048 截断)
造成,也可能是单层 head 的结构性上限——**这正是待验证的关键分叉**。

---

## 4. 判决与关键未决问题

| 维度 | 状态 |
|---|---|
| 信号存在性 | ✅ 强(AUC 0.86,校准干净) |
| 长 run 存在性(强 drafter) | ✅(1.5B 20% 吃满) |
| 长 run 存在性(EAGLE head) | ❌(短跑者) |
| 能否超越 vanilla EAGLE-3 | ❓ **未证实** |
| 作为论文主结果 | ❌ 还不够 |

**关键未决问题**:一个**训得好的** EAGLE head 会不会恢复长 run?
- 若会 → 便宜草稿 + 自适应深度 = 高接受长度 + 低草稿成本,**同时超越 EAGLE 固定链与 UMbreLLa 树**,
  method 成立。
- 若不会(单层 head 结构性短视) → 需换 method 杠杆(级联 drafter / 树 / 更深 head)。

**下一步实验**(定生死):训一个强 head(更多数据、不截断)→ 大 cap 采集其**置信-接受**逐 token 数据
→ 量化自适应深度 vs 最优固定深度的 committed-per-forward → 若 >10% 增益,落进解码循环测真实 tok/s。
